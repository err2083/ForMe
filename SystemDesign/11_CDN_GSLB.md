# CDN 및 GSLB를 활용한 글로벌 시스템 설계

1. 핵심 개념 정의
    - CDN (Content Delivery Network): 전 세계에 분산된 서버를 통해 콘텐츠(정적 리소스 등)를 사용자에게 신속하게 전달하는 기술입니다
    - GSLB (Global Server Load Balancing): 여러 지리적 지역에 위치한 서버 간의 트래픽을 최적화하여 서비스 안정성과 성능을 조정하는 기술입니다
2. 글로벌 시스템 요구사항 (사례: constsite.com)
    - 전 세계 건설 현장에서 사용하는 시스템을 가정한 설계 요구사항입니다
    - 통일성: 모든 사이트에서 동일한 시스템 로직 사용
    - 독립성: 각 현장은 자체 데이터베이스를 관리
    - 네트워크 제약: 매우 느리거나 불안정한 연결(최대 2초 레이턴시, 20% 패킷 손실 허용) 환경에서도 본사 접속 없이 동작 가능해야 함
    - 데이터 무결성: 비즈니스 중요 데이터의 정확성 유지 필수
3. 지역별 최적화를 위한 주요 기술
    - 서로 다른 지역의 사용자에게 동일한 도메인으로 최적의 서버를 연결하기 위해 다음 기술이 사용됩니다
    - Anycast Network: 전 세계 여러 서버에 동일한 IP 주소를 부여하고, 라우팅 테이블을 통해 사용자와 물리적으로 가장 가까운 서버로 요청을 자동 전달합니다
    - GeoDNS: 사용자의 IP 주소를 기반으로 가장 가까운 엣지 서버의 IP를 반환합니다
    - BGP (Border Gateway Protocol): 네트워크 경로를 최적화하여 패킷이 가장 효율적인 경로로 이동하게 합니다
4. GSLB 아키텍처 및 동작 흐름
    - GSLB는 DNS 쿼리 단계에서 개입하여 최적의 서버 IP를 결정합니다
    - 동작 순서 (GSLB Flow)
        - 클라이언트가 로컬 DNS에 도메인 IP를 요청합니다.
        - 로컬 DNS가 상위 DNS 서버에 쿼리하고, 상위 서버는 GSLB 정보를 반환(권한 위임)합니다.
        - 로컬 DNS가 GSLB에 다시 IP를 요청합니다.
        - GSLB는 서버 상태, 부하, 사용자 위치 등을 고려해 최적의 서버 IP를 로컬 DNS에 보냅니다.
        - 로컬 DNS는 이 IP를 캐시하고 클라이언트에게 전달합니다.
    - GSLB 주요 라우팅 규칙
        - Geolocation: 사용자 지역(예: 미국, 캐나다)에 따른 라우팅.
        - Health Check: 서버 상태가 'UP'인 경우에만 트래픽 전송.
        - Load-Based: 서버 부하(예: 80% 미만)에 따른 분산.
        - Bandwidth/Latency-Based: 네트워크 대역폭 사용량이나 지연 시간이 낮은 서버 선택.
        - User Profile: VIP 유저 등을 전용 서버로 안내.
        - Disaster Recovery (DR): 주 서버 다운 시 예비 서버로 자동 전환.
5. CDN 동작 및 콘텐츠 처리
    - 정적 콘텐츠 (Static): JS, HTML, 이미지 등은 CDN 엣지 서버에 캐시되어 원본 서버 부하를 줄이고 로딩 속도를 높입니다
    - 동적 콘텐츠 (Dynamic): 실시간 처리가 필요한 API 요청 등은 GSLB를 통해 가장 가까운 지역의 API 서버와 DB로 연결됩니다
    - 캐시 미스 (Cache Miss): 엣지에 없는 데이터는 오리진 서버에서 가져와 캐시한 후 사용자에게 전달합니다
6. 데이터 및 파일 동기화 전략
    - 데이터베이스 동기화
        - 방식: MySQL 바이너리 로그 복제, PostgreSQL/MongoDB 복제 솔루션, Oracle GoldenGate(OGG) 등 활용.
        - 데이터 분류:
            - Master/Metadata/Reference: 주로 본사에서 현장으로 양방향 또는 단방향 동기화.
            - Event/Stats: 현장 발생 데이터를 본사로 수집(원웨이 싱크).
    - 파일 동기화 (File Sync)
        - 도전 과제: 낮은 대역폭과 높은 레이턴시 환경에서 대용량 파일 전송의 어려움.
        - 해결 방안: 파일 압축, 분할 전송, 실패 시 재시도 기능 필수.
        - 도구: rsync의 --bwlimit 옵션 등을 사용하여 네트워크 점유율을 조절하며 동기화 수행.
7. 최적화 요약
    - 클라이언트 사이드: 데이터 압축, 배치 처리, 비동기 작업(화면 멈춤 방지), 로컬/세션 스토리지 활용
    - 서버 사이드: Anycast DNS 및 BGP 라우팅 구성, 메시지 큐 등을 통한 비동기 처리, 지속적인 모니터링
