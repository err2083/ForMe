# Redis Set 타입의 특징과 활용

## 1. Redis Set 개요 및 특징

Redis Set은 중복을 허용하지 않는 문자열 값들의 **무순서(Unordered) 집합**입니다. 수학의 집합 개념을 컴퓨터 구조로 구현한 자료구조입니다.

* **고유성(Uniqueness)**: 동일한 값을 여러 번 추가해도 실제로는 단 하나의 값만 저장됩니다. 유일한 값만 관리해야 할 때 유용합니다.
* **무순서성(Unordered)**: 요소를 추가한 순서를 유지하지 않습니다. 조회 시 매번 다른 순서로 결과가 반환될 수 있으므로 순서에 의존하는 로직에는 적합하지 않습니다.
* **멤버십 체크(O1)**: 특정 값이 집합에 포함되어 있는지 확인하는 작업이 매우 빠릅니다(O(1)). 이는 리스트와 차별화되는 가장 큰 장점입니다.
* **집합 연산 지원**: 여러 개의 Set 간에 교집합, 합집합, 차집합 연산을 서버 측에서 직접 수행할 수 있습니다.

## 2. 내부 저장 구조 및 인코딩

데이터의 특성과 크기에 따라 두 가지 방식으로 내부 데이터를 인코딩하여 메모리를 효율적으로 관리합니다.

* **Int Set (정수 집합)**:
  * **조건**: 모든 요소가 64비트 정수이고, 요소의 개수가 설정된 기준(set-max-intset-entries)보다 적을 때 사용합니다.
  * **특징**: 정렬된 정수 배열 구조로 메모리 효율이 매우 좋습니다. 이진 탐색을 통해 O(log N)의 복잡도로 검색합니다.
* **Hash Table (해시 테이블)**:
  * **조건**: 요소에 문자열이 포함되거나 요소의 개수가 많아질 때 자동으로 전환됩니다.
  * **특징**: 해시 함수를 사용하여 각 요소를 버킷에 매핑합니다. 검색, 삽입, 삭제에 대해 평균 O(1)의 매우 빠른 성능을 보입니다.
  * **주의**: 각 요소마다 해시 테이블 오버헤드가 발생하므로 리스트에 비해 메모리 사용량이 상대적으로 많습니다.

## 3. 핵심 명령어 상세

### 1) 요소 관리 (추가 및 삭제)

* **SADD key member [members...]**: 하나 이상의 요소를 추가합니다.
  * 이미 존재하는 값은 무시하며, **실제로 새로 추가된 요소의 개수**를 반환합니다.
  * 시간 복잡도는 추가하려는 요소의 개수(N)에 비례하며, 기존 Set의 크기와는 무관합니다.
* **SREM key member [members...]**: 하나 이상의 요소를 삭제합니다.
  * 존재하는 요소만 삭제하고, **실제로 삭제된 요소의 개수**를 반환합니다.
  * 리스트와 마찬가지로 마지막 요소가 삭제되어 Set이 비게 되면 해당 키도 자동으로 메모리에서 삭제됩니다.

### 2) 조회 및 확인

* **SMEMBERS key**: Set에 포함된 모든 요소를 배열 형태로 반환합니다.
  * **주의**: 시간 복잡도가 O(N)(N은 Set의 크기)이므로, 요소가 매우 많은 대용량 Set에서 실행 시 Redis의 싱글 스레드를 점유하여 다른 작업을 방해할 수 있습니다. 큰 Set은 `SSCAN` 명령어를 권장합니다.
* **SISMEMBER key member**: 특정 값이 Set에 존재하는지 확인합니다.
  * 존재하면 1, 없으면 0을 반환하며 O(1)로 매우 빠릅니다.
* **SMISMEMBER key member [members...] (6.2+)**: 여러 개의 값이 존재하는지 한 번에 확인합니다.

## 4. 집합 연산 (Set Operations)

두 개 이상의 Set을 비교하여 결과를 도출합니다. 실무에서는 특수한 통계 작업 외에는 사용 빈도가 낮을 수 있으나, 네트워크 전송량을 줄이는 데 효과적입니다.

* **SINTER key [key...] (교집합)**: 지정한 모든 Set에 공통으로 포함된 요소만 반환합니다. (AND 연산)
* **SUNION key [key...] (합집합)**: 지정한 모든 Set의 요소를 합치며 중복을 제거하여 반환합니다. (OR 연산)
* **SDIFF key [key...] (차집합)**: 첫 번째 Set에는 존재하지만, 이후 나열된 다른 Set들에는 없는 요소만 반환합니다. (MINUS 연산)

## 5. 실무 활용 및 성능 고려사항

* **멤버십 체크**: 리스트는 특정 값을 찾기 위해 전체를 순회해야 하지만, Set은 즉시 확인이 가능하므로 "이미 처리된 데이터인가?"를 판별하는 용도로 최적입니다.
* **방문자 수 집계**: 중복을 자동으로 제거하므로 특정 기간의 유니크 방문자(MAU, DAU)를 계산할 때 효과적으로 사용됩니다.
* **성능 관리**: `SMEMBERS`와 같은 O(N) 명령어는 데이터가 커질수록 위험하므로, 실무에서는 커서 기반의 `SSCAN`을 사용하여 나누어 조회하는 것이 안전합니다.
