# Redis Sorted Set의 특징과 실무 활용

## 1. Redis Sorted Set 개요

Sorted Set(ZSET)은 중복되지 않는 요소(Member)를 저장하면서, 각 요소마다 가중치인 **점수(Score)**를 부여하여 자동으로 정렬 상태를 유지하는 자료구조입니다.

* **구성 요소**:
  * **멤버(Member)**: 실제 저장되는 고유한 문자열 값 (Set과 동일하게 중복 불가).
  * **점수(Score)**: 정렬의 기준이 되는 부동 소수점(숫자) 값.
* **정렬 방식**: Score를 기준으로 오름차순 자동 정렬됩니다. 만약 Score가 같다면 멤버의 사전 순(Lexicographical)으로 정렬됩니다.
* **성능**: 삽입과 삭제 시 **O(log N)**의 시간 복잡도를 가지며, 대용량 데이터에서도 매우 효율적인 조회가 가능합니다.
* **활용 사례**: 게임 리더보드(전투력 순위), 월별 거래 금액 순위 등 실시간 랭킹 시스템에 최적화되어 있습니다.

## 2. 내부 저장 구조 및 인코딩

데이터의 규모에 따라 메모리 효율과 성능을 조절하기 위해 두 가지 방식을 사용합니다.

### 1) ListPack (리스트팩)

* 데이터 양이 적고 점수/멤버의 크기가 작을 때 사용하는 압축된 형태입니다.

### 2) Skip List + Hash Table (결합 구조)

데이터가 일정 기준을 넘어서면 검색 효율을 위해 두 가지 구조를 결합하여 사용합니다.

* **Skip List**: 여러 레벨의 포인터를 유지하여 **O(log N)** 수준의 빠른 범위 검색을 지원합니다. (점수 기반 조회에 사용)
* **Hash Table**: 멤버 이름을 기반으로 특정 요소의 존재 여부나 점수를 즉시 확인합니다. (멤버 기반 조회에 사용)
* **메모리 고려사항**: 이중 구조를 유지하고 여러 레벨의 포인터를 저장하기 때문에, 일반적인 Set에 비해 약 2~3배 더 많은 메모리를 사용합니다.

## 3. 핵심 명령어 상세

### 1) 요소 추가 및 업데이트

* **ZADD key [options] score member [score member...]**: 하나 이상의 요소를 추가하거나 기존 요소의 점수를 업데이트합니다.
  * **주요 옵션**:
    * **NX**: 멤버가 존재하지 않을 때만 추가합니다.
    * **XX**: 멤버가 이미 존재할 때만 점수를 업데이트합니다.
    * **GT**: 새로운 점수가 기존 점수보다 클 때만 업데이트합니다.
    * **LT**: 새로운 점수가 기존 점수보다 작을 때만 업데이트합니다.
    * **CH**: 추가된 요소뿐만 아니라 점수가 변경된 요소의 개수까지 반환하도록 합니다.

### 2) 범위 및 순위 조회

* **ZRANGE key start stop [WITHSCORES]**: 순위(인덱스) 범위를 기준으로 **오름차순** 조회합니다. (0은 가장 낮은 점수, -1은 마지막 요소)
* **ZREVRANGE key start stop [WITHSCORES]**: **내림차순**으로 범위를 조회합니다. 상위 랭킹을 보여줄 때 유용합니다.
* **ZRANK key member**: 특정 멤버의 **오름차순 순위**를 반환합니다. (0부터 시작)
* **ZREVRANK key member**: 특정 멤버의 **내림차순 순위**를 반환합니다. 리더보드에서 '나의 등수'를 표시할 때 주로 사용합니다.

### 3) 점수 변경 및 삭제

* **ZINCRBY key increment member**: 특정 멤버의 점수를 지정한 만큼 증가시키거나 감소시킵니다. (원자적 연산으로 Race Condition 방지)
* **ZREM key member [member...]**: 하나 이상의 멤버를 삭제하고 실제 삭제된 개수를 반환합니다.
* **ZREMRANGEBYRANK key start stop**: 순위 범위를 지정하여 해당 구간의 요소를 한 번에 삭제합니다.

## 4. 실무 가이드 및 주의사항

* **자동 생성 및 삭제**: 키나 멤버가 없으면 자동으로 생성하며, 마지막 요소가 삭제되어 비어 있게 되면 Redis가 해당 키를 자동으로 메모리에서 제거합니다.
* **반환 타입**: Redis 내부에서 점수는 숫자로 관리되지만, 결과 반환 시에는 문자열 형태로 전달됩니다 (예: "100", "100.5").
* **메모리 관리**: Sorted Set은 성능을 위해 메모리를 상대적으로 많이 소요하므로, 42억 개(2^32-1)까지 저장 가능하다는 이론적 수치보다는 서버의 가용 RAM 용량을 상시 모니터링해야 합니다.
